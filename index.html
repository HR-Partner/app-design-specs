Introduction

Welcome to the HR Partner Portal Style Guide. This comprehensive reference documents all design patterns, components, and utilities used in the employee self-service portal.

How to Use This Guide

  • • Use the theme switcher above to preview all 6 color themes
  • • Click the Copy button on code examples to copy HAML snippets
  • • Use the search box to quickly find specific components or patterns
  • • All examples are live and reflect the currently selected theme

Regenerating This Guide

After making changes to portal components, regenerate this guide with:

bundle exec rake generate_portal_style_guide

Responsive Design

Mobile-First Approach

The portal uses a mobile-first approach. Styles are applied to mobile by default, with responsive modifiers adding complexity for larger screens. 

/ Hidden on mobile, visible on desktop
.hidden.md:block

/ 1 column mobile, 2 columns tablet, 3 columns desktop
.grid.grid-cols-1.md:grid-cols-2.lg:grid-cols-3

/ Smaller text on mobile, larger on desktop
.text-base.md:text-lg

Common Patterns

  • • Sidebar: Fixed on desktop (md:+), overlay drawer on mobile
  • • Navigation: Hamburger menu on mobile, always visible on desktop
  • • Grids: Single column mobile → multi-column desktop
  • • Typography: Smaller sizes on mobile for better readability
  • • Spacing: Tighter padding/margins on mobile

Accessibility

ARIA Attributes

  • • role="dialog" - Modal elements
  • • aria-modal="true" - Modal indicators
  • • aria-labelledby - Label associations
  • • aria-expanded - Toggle states
  • • aria-hidden - Hidden decorative content

Semantic HTML

  • • Use <button> for buttons (not div or a)
  • • Use <label for="id"> for form labels
  • • Use <nav> for navigation
  • • Maintain proper heading hierarchy (h1 → h2 → h3)
  • • Use <fieldset> and <legend> for form groups

Focus Management

  • • All interactive elements have visible focus rings
  • • Use focus:ring-2 pattern
  • • Maintain logical tab order
  • • Trap focus within modals when open
